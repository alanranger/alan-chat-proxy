ℹ️ What the Numbers Mean

  • Content Imported: Rows read from your CSVs (input size by category).
  • Web Scraping: Unique URLs queued for Step 2 based on selected sources; duplicates across sources are removed here.
  • Success / Failed / Stopped / Skipped: Outcome of Step 2 for this run (Success means text saved to page_chunks and JSON‑LD merged into page_entities).
  • CSV Count (per category): Rows imported from CSVs in Step 1 for that category label.
  • Duplicates Removed (per category): URLs eliminated from the scrape queue because they appeared more than once across sources. This is de‑dupe of the queue, not database duplicates.
Where data goes:
  • page_chunks: Plain‑text chunks per URL (replaced on each successful scrape).
  • page_entities: Normalized JSON‑LD entities {event, product, article, service}. Existing (url, kind) are always updated so rescheduled events refresh.
  • event_product_links_auto: Created by Step 3 (Run Mapping).
  • Views used by chat/analytics: v_events_for_chat, v_service_content, v_product_content, v_blog_content. “Finalize” warms/refreshes these.
